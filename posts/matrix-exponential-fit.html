Learning Linear Dynamics

Four ways to recover A from observations of z(t) = exp(At)z₀

Author

Etienne Meunier

Published

June 30, 2026

The inverse problem

In the previous post we studied how to evaluate \(e^{At}z_0\) efficiently. Here we flip the question:

Given observations \(z(t_1), \ldots, z(t_n)\) of a trajectory driven by \(\dot{z} = Az\), recover the matrix \(A\).

This is the core sub-problem in system identification, Koopman operator learning, and physics-informed machine learning. We compare four approaches that differ in their data requirements, computational cost, and sensitivity to noise.

Ground truth

We use a stable node with off-diagonal coupling as the true system and observe the trajectory at \(n = 4\) uniformly-spaced time points. The eigenvalues are \(-0.1\) and \(-1\) — real, stable, with well-separated time scales.

\[A = \begin{pmatrix}-0.4 & 0.6 \\ 0.3 & -0.7\end{pmatrix}, \quad z_0 = \begin{pmatrix}1 \\ 1.5\end{pmatrix}\]

def _streamplot(ax, M, color, alpha=0.45, density=0.9,
                xlim=(-0.2, 1.7), ylim=(-0.2, 1.9), n_grid=20):
    M_np = M.real.detach().numpy() if torch.is_complex(M) else M.detach().numpy()
    xs = np.linspace(*xlim, n_grid)
    ys = np.linspace(*ylim, n_grid)
    X, Y = np.meshgrid(xs, ys)
    pts = np.stack([X.ravel(), Y.ravel()], axis=1)
    UV = pts @ M_np.T
    sp = ax.streamplot(xs, ys,
                       UV[:, 0].reshape(n_grid, n_grid),
                       UV[:, 1].reshape(n_grid, n_grid),
                       density=density, color=color,
                       linewidth=0.7, arrowsize=0.6)
    sp.lines.set_alpha(alpha)
    sp.arrows.set_alpha(alpha)
    ax.set_xlim(xlim); ax.set_ylim(ylim)
    ax.set_aspect('equal')

Approach 1 — Gradient descent on matrix exponential

Idea

Directly optimise \(B \in \mathbb{R}^{p \times p}\) to minimise the squared error at every observed time:

\[\mathcal{L}(B) = \sum_{i=1}^{n} \left\| e^{B t_i} z_0 - z(t_i) \right\|^2\]

Gradients flow through torch.matrix_exp (which uses a Padé approximation internally). No structure is imposed on \(B\) — stability, symmetry, sparsity are unconstrained.

Data requirement: a single trajectory.

Code

B = torch.randn((2,2), dtype=torch.float64, requires_grad=True)
opt = torch.optim.SGD([B], lr=0.005)

def step() :
    opt.zero_grad()
    z_pred = torch.matrix_exp(B[None]*t_gt[:, None, None])@z0
    L = (z_pred - z_gt).pow(2).sum()
    L.backward()
    opt.step()
    return {'loss' : L.item(), 'forb' : (A - B).pow(2).sum()}

metrics = [step() for i in tqdm(range(1000))]
plot 
with torch.no_grad() :
    z_pred = torch.matrix_exp(B[None]*t_gt[:, None, None])@z0
    plot_results(metrics, B, z_pred, z_gt, t_gt, title='Approach 1 — Gradient descent on expm')

Approach 2 — Neural ODE

Idea

Same loss as Approach 1, but the forward pass uses a numerical ODE solver instead of the closed-form matrix exponential:

\[\mathcal{L}(B) = \sum_{i=1}^{n} \left\| \texttt{odeint}(B,\, z_0,\, t_i) - z(t_i) \right\|^2\]

Gradients propagate through the integrator via the adjoint method. The two approaches are equivalent in theory but differ numerically: odeint introduces discretisation error that can smooth the loss landscape.

Data requirement: a single trajectory.

Code

B = torch.randn((2,2), dtype=torch.float64, requires_grad=True)
opt = torch.optim.Adam([B], lr=0.01)

def step() :
    opt.zero_grad()
    z_pred = odeint(lambda t, x : B@x, z0, t_gt)
    L = (z_pred - z_gt).pow(2).sum()
    L.backward()
    opt.step()
    return {'loss' : L.item(), 'forb' : (A - B).pow(2).sum()}

metrics = [step() for i in tqdm(range(2000))]
plot 
with torch.no_grad() :
    z_pred = odeint(lambda t, x : B@x, z0, t_gt)
    plot_results(metrics, B, z_pred, z_gt, t_gt, title='Approach 2 — Neural ODE')

Approach 3 — Derivative matching

Idea

Since \(\dot{z}(t) = Az(t)\) holds pointwise, stacking observations column-wise gives a linear equation in \(A\):

\[\dot{Z} = A Z, \qquad Z = \bigl[z(t_1) \;\cdots\; z(t_n)\bigr] \in \mathbb{R}^{p \times n}\]

The least-squares solution is:

\[B = \dot{\hat{Z}}\, Z^{\dagger}\]

where \(Z^{\dagger} = Z^\top(ZZ^\top)^{-1}\) and \(\dot{\hat{Z}}\) is estimated by differentiating a cubic spline fitted to the observations. No gradient descent is required — this is a single linear solve.

The quality of the estimate depends on how faithfully the spline captures \(\dot{z}\), which degrades as observations become sparser.

Data requirement: a single trajectory.

Approach 4 — Matrix logarithm

Idea

With \(K\) trajectories from different initial conditions, all observed at the same time \(t\), stack them column-wise:

\[Z(t) = e^{At} Z_0, \qquad Z_0 = \bigl[z_0^{(1)} \;\cdots\; z_0^{(K)}\bigr] \in \mathbb{R}^{p \times K}\]

For \(K \geq p\) with \(Z_0\) full row rank, the transition matrix \(e^{At}\) is identified by least squares, and \(A\) is recovered via the matrix logarithm:

\[e^{At} = Z(t)\, Z_0^{\dagger} \implies A = \frac{1}{t}\,\operatorname{logm}\!\bigl(Z(t)\, Z_0^{\dagger}\bigr)\]

This is a closed-form, one-shot estimate — no gradient descent, no derivative estimation.

WarningBranch cut

logm applies the principal matrix logarithm, which recovers \(\lambda_k t\) from \(e^{\lambda_k t}\) only when \(|\operatorname{Im}(\lambda_k t)| < \pi\). For oscillatory modes with frequency \(\omega\) this requires \(\omega t < \pi\). Use a small observation time \(t\) to stay on the correct branch.

Data requirement: \(K \geq p\) trajectories from linearly independent initial conditions.

Code

Approach 5 : find the closest diagonal dynamic

One can assume that the dynamic is diagonal (each component is independent).

In this case:

\[ z_{k,p}(t) = e^{\lambda_p t}\, z_{k,p}(0) \qquad \forall\, t,\, k \]

Thus this amounts to solving:

\[ \lambda_p = \underset{\lambda_p}{\operatorname{argmin}} \sum_{k \in \mathcal{S}} \sum_{i} \bigl(z_{k,p}(t_i) - e^{\lambda_p t_i} z_{k,p}(0)\bigr)^2 \]

Taking the log of the model linearises the problem in \(\lambda_p\):

\[ \lambda_p = \underset{\lambda_p}{\operatorname{argmin}} \sum_{k \in \mathcal{S}} \sum_{i} \bigl(\log z_{k,p}(t_i) - \lambda_p\, t_i - \log z_{k,p}(0)\bigr)^2 \]

We can solve this in matrix form. Stacking all time points into \(\mathbf{t} \in \mathbb{R}^{n \times 1}\) and defining \(L_{i,p} = \log z_p(t_i) - \log z_p(0)\):

\[ \mathbf{t}\,[\lambda_1, \ldots, \lambda_p] = L \implies \Lambda = \mathbf{t}^{\dagger}\, L \]

if (z_gt > 0).all() and (z0 > 0).all():
    l_gt = torch.log(z_gt)
    l_0  = torch.log(z0)
    lbd  = (torch.linalg.pinv(t_gt[:, None].to(torch.float64)) @ (l_gt - l_0)).squeeze()
    B    = torch.diag(lbd)
else:
    print("Approach 5 not applicable: spiral trajectory goes negative — log undefined.")
    B = torch.full((2, 2), float('nan'), dtype=torch.float64)

metrics = {}
plot 
with torch.no_grad() :
    z_pred = odeint(lambda t, x : B@x, z0, t_gt)
    plot_results(metrics, B, z_pred, z_gt, t_gt, title='Approach 5 — Diagonal log regression')

Comparing approaches

We run all five methods on the three canonical linear systems from the previous post and measure how well each recovered \(B\) approximates \(A\) in Frobenius norm \(\|A - B\|_F\). Approaches 1 and 2 use Adam (200 epochs); approach 4 uses \(K = 20\) independent trajectories from random initial conditions; all others use the same 8 observations over \([0, 2]\).

\[ A_{\text{stable node}} = \begin{pmatrix}-2 & 0 \\ 0 & -0.5\end{pmatrix}, \qquad A_{\text{center}} = \begin{pmatrix}0 & -2 \\ 2 & 0\end{pmatrix}, \qquad A_{\text{degen. node}} = \begin{pmatrix}-1 & 1 \\ 0 & -1\end{pmatrix} \]

def _fit1(z0, t, z_obs, epochs=200):
    B   = torch.zeros(2, 2, dtype=torch.float64, requires_grad=True)
    opt = torch.optim.Adam([B], lr=0.02)
    for _ in range(epochs):
        opt.zero_grad()
        (torch.matrix_exp(B[None]*t[:,None,None])@z0 - z_obs).pow(2).sum().backward()
        opt.step()
    return B.detach()

def _fit2(z0, t, z_obs, epochs=200):
    B   = torch.zeros(2, 2, dtype=torch.float64, requires_grad=True)
    opt = torch.optim.Adam([B], lr=0.02)
    for _ in range(epochs):
        opt.zero_grad()
        (odeint(lambda s, x: B@x, z0, t) - z_obs).pow(2).sum().backward()
        opt.step()
    return B.detach()

def _fit3(z0, t, z_obs):
    sp = CubicSpline(t.numpy(), z_obs.numpy())
    return (torch.tensor(sp(t.numpy(), 1)).T) @ torch.linalg.pinv(z_obs.T)

def _fit4(A_true, T=0.02, K=20):
    Z0 = torch.randn(2, K, dtype=torch.float64)
    Z1 = torch.matrix_exp(A_true * T) @ Z0
    return torch.tensor(logm((Z1 @ torch.linalg.pinv(Z0)).numpy()).real / T)

def _fit5(z0, t, z_obs):
    if not ((z_obs > 0).all() and (z0 > 0).all()):
        return None
    l_diff = torch.log(z_obs) - torch.log(z0)
    lbd    = (torch.linalg.pinv(t[:,None]) @ l_diff).squeeze()
    return torch.diag(lbd)

Summary

Approach Closed form Data needed Main sensitivity
Gradient on expm No Single trajectory Loss landscape, learning rate
Neural ODE No Single trajectory Solver step size, learning rate
Derivative matching Yes Single trajectory Spline quality (sparse data)
Matrix logarithm Yes \(K \geq p\) trajectories, same \(t\) Branch cut (use small \(t\))
Diagonal log regression Yes Single trajectory Assumes \(A\) diagonal; log-space loss

The closed-form approaches (3, 4, 5) are generally preferable when their assumptions hold: derivative matching for a single dense trajectory, matrix logarithm when you control initial conditions, and diagonal regression when the dynamics are known to be decoupled.
